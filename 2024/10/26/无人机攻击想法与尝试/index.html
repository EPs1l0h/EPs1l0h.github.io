无人机攻击想法与尝试

拟态的比赛有一个之前从来没有在任何比赛出现过的项目：现场给你一台无人机，主办方给你提供 hackrf + 可以抓包的无线网卡，让你去打无人机（商用无人机实网挑战）

以前从来没接触过相关的知识，研究的有点慢。借了实验室一套 Hackrf 开发板和一根外接天线，买了一个 JYMind 03 的无人机进行了一些攻击尝试，希望能给以后相关的比赛做一点技术储备

GPS 面的攻击

一般就是 GPS 欺骗。做到这一点的话，有几种攻击思路：

  1. 欺骗无人机的起点，待无人机起飞后 GPS 定向返航就会往虚假地点返航；
  2. 在飞行过程中给一个假的 GPS 信号，让无人机以为自己超过最大航距然后返航；
  3. 给一个在禁飞区的 GPS 定位，比较高级的无人机就会识别到禁飞区然后迫降。

一次 GPS 欺骗实验

对于 Hackrf One，需要一个生成卫星信号的gps-sdr-sim和用于发射信号的hackrf_transfer，首先得知道模拟的地方的经纬度，可以在 Google Map上拿坐标，右键一下就行。Google 的卫星地图使用的是 WGS84 坐标，和 gps-sdr-sim 用的是一个格式，不用额外转换。

这里我们选择天安门作为我们要发送的坐标（妥妥的禁飞区）

然后需要在 NASA 的网站 https://cddis.nasa.gov/archive/gnss/data/daily/2024/ 去找比较新 GPS 星历。它包含了GPS卫星的位置、速度、时钟等信息，用来生成假的 GPS 信号。

这里我们选择 https://cddis.nasa.gov/archive/gnss/data/daily/2024/318/24n/brdc3180.24n.gz，也就是 2024年11月13日的最新星历，然后使用gps-sdr-sim：./gps-sdr-sim -e brdc3180.24n -l 39.908741,116.397480,100 -b 8 ，

-l后面是经纬度和海拔（注意不能有空格），-b是数据位数，hackrf 支持的是 8 位，-e 是指定星历，默认生成 300 秒的信号，结果会默认存在gpssim.bin文件中

随后用 hackrf 调制发送：hackrf_transfer -t gpssim.bin -f 1575420000 -s 2600000 -a 1 -x 10

-t 是 GPS 信号文件，-f 是民用 GPS 卫星频率 1575.42MHz，-s 是采样率，-a 是 TX RF 放大器（外接天线），-x 是 TX VGA (IF) 增益

手机和无人机的信号都能进行欺骗，但是不知道是不是咱们实验室翻出来的这块 hackrf 的板子有点问题，或者是实验室翻出来的外接天线的功率有问题之类的，欺骗成功率比较低，得试很多次才成功一次，此外，重启手机可以有效提高欺骗成功率。

可以看出，欺骗成功后，手机已经显示其坐标位于我们发送的天安门坐标，39.90873357804588, 116.39748764719477

另一个问题是北邮附近禁飞，想 试对无人机最终的攻击效果得去比较远的地方尝试

射频面的攻击

无人机会配套一个遥控器去进行控制。不同厂家生产的遥控器所在频段也不是很一样，这个可以参考说明书/官方网站/店铺的公开参数去查大概的频段。大部分的无人机遥控器频段都在 2.4GHz。

确定信号频段

使用 gqrx + hackrf 做到音频+可视化界面一起监听无线电信号，设置好后在 2.4GHz +- 100MHz 的频段去尝试监听。

把遥控器放在 hackrf 的外接天线旁边，做一些开关动作，找到有明显契合该动作的无线电变化的频段，就是遥控器的频段。经过一晚上人肉遍历以后，发现2.478Ghz这个频段特征非常明显，随着遥控器开关，该频段会随之显示信号/消失。

重放攻击

这里衍生出几种可能的攻击。一种是重放攻击，如果对重放攻击防护的不是很完善，可以首先录制 RC 和无人机建立连接的握手通信，然后再录制起飞、迫降等操作的遥控通信，随后分别进行重放，劫持无人机控制权并且进行各种遥控操作。可惜我自己去尝试的时候没有成功，可能是有对重放攻击的校验，也可能是天线发送功率有问题。

这里涉及到三个概念：固定码，学习码和滚动码

固定码：固定码的地址码是固定不变的，通常由一组预设的8位或更长的地址码组成。每个遥控器和接收器都有一个唯一的地址码，用于标识发送和接收的设备。

学习码：学习码是一种通过“学习”配对的编码系统。在这种系统中，遥控器和接收器之间的通信信号并不是通过固定的预设码传输，而是通过一个“学习”过程使接收器记住特定遥控器发出的码。遥控器和接收器在首次配对时，通过一定的学习过程建立起匹配关系，只有通过这个学习过程配对的遥控器才会与接收器进行通信。

学习的过程为：

  1. 接收器通过一些物理按钮进入学习模式，指示灯闪烁
  2. 遥控器发送信号（可能包含一个动态生成的编码）
  3. 接收器接收并存储信号，同时确认存储，指示灯常亮
  4. 完成配对并退出学习模式

滚动码：滚动码系统在每次发送时都会生成一个新的、唯一的码。每个码都是基于当前的计数值（同步计数）和一个固定的密钥（通常是加密算法）生成的。同步计数会随着每次发送而递增，因此即使是同一个按键被多次按下，每次发送的码也都不同。

录制信号：

使用指令进行信号录制并存入文件，-r 为写入文件，dian.raw 即为录制的文件名；-f 为采样频率； -g 为 RX VGA(baseband) gain；-l 为 RX LAN(IF) gain。-g -l 用来进行信号放大。运行后对准天线按下遥控器，可以多按几下，然后ctrl+c停止录制。

hackrf_transfer -r dian.raw -f 2478000000 -g 16 -l 32

重放信号

录制完毕并存入文件后，使用下述指令进行重放，重放的文件就是刚刚录制的。-t 用来制定文件；-x 为 TX VGA (IF) gain 放大信号。重放时天线对准电车，即可实现车辆解锁。

hackrf_transfer -t dian.raw -f 2478000000 -x 47

噪声覆盖

另一种是直接干扰遥控器的控制信号。因为之前已经监听到遥控器的频段，所以直接在这个频段用 hackrf 去不停地发送噪音去填满这个频段，就可以阻碍无人机和遥控器的通信了。

用 hackrf 的 Jammer 模块，中心频段设为 2478MHz，宽度设个 2MHz，跳频 10kHz，每一跳隔 50ms，然后开始发送噪音。我一直觉得借的这根天线可能有点问题的地方就在于，这里对于 hackrf 的操作应该没啥问题，并且找了很多用这种方法攻击的资料，它们甚至可以阻碍 2.4GHz 的 WiFi 通信，但是我这里并没有成功阻碍遥控器和无人机的通信。具体失败的原因还没分析出来。

WiFi 面的攻击

很多无人机都支持 WiFi 连接手机的飞控软件进行控制，所以 WiFi 也是一个攻击面。

这个需要用到一个无线网卡，由于之前买的 rtl8812au 的网卡没有找到既能用又支持监控模式的 linux 驱动，所以重新买了个 rtl8812bu 的，还没到货，所以这里都是一些概念性的攻击手段，没有具体去验证。

如果没有绑 MAC 地址的话，用常规的 WiFi 渗透思路通过发包让对方的飞控掉线，然后重新用我自己的客户端连接并接管无人机理论上是可行的。这里可以用到 aircrack-ng 套件，打开网卡的监听模式，用 airodump-ng 扫描到无人机的 WiFi 信号以及其 MAC 地址，然后用 aireplay-ng 循环发包进行攻击，将正在连接该 WiFi 的设备挤下线，然后我们自己登录进该 WiFi 完成接管。

另一种方法就是打开监控模式后通过网卡抓包，分析无人机的通信协议，然后去构造起飞/迫降等操作的合法流量包发送给无人机，迫使其做出一些行动。

通信中断

通过抓包可以看到无人机在回传视频流时使用UDP协议，发送在 62513，接收端口 7777，可以考虑向目标端口高频发包，导致其通信中断

抓包

如果直接用手机添加抓包软件的系统证书，然后和无人机进行连接来抓包，测试发现无法解密HTTPS信息，存在SSLPinning防止中间人攻击。

但是可以抓到UDP广播的包，用手机B连接无人机wifi，并打开手机热点，用手机A连接B的热点，并把抓包软件的证书挂载到系统目录，可以绕过SSLPinning，抓到手机B和无人机的流量。（看起来好像是因为这个无人机提供的图片和视频资源在回传的时候是广播的，这个接口只要在内网里就可以访问然后对无人机的图片和视频进行下载）
